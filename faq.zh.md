# Frequently Asked Questions

## What is WebTorrent?

**网络流**是第一个在**浏览器**. 是的，没错。浏览器。

它完全用网络语言javascript编写，并使用**韦伯特**真正的点对点传输。不需要浏览器插件、扩展或安装。

使用开放的Web标准，WebTorrent将网站用户连接在一起，形成一个分布式的、分散的浏览器到浏览器网络，以实现高效的文件传输。

## Why is this cool?

想象一下像YouTube这样的视频网站，**访问者帮助宿主网站内容**. 使用WebTorrent支持的网站的用户越多，网站的速度越快，恢复能力越强。

浏览器到浏览器的通信**排除中间人**让人们用自己的方式交流。没有更多的客户机/服务器——只是一个对等网络。WebTorrent是通往[再分散][redecentralize]网络。

> 我们对网络的编码方式将决定我们的在线生活方式。所以我们需要将我们的值烘焙到代码中。表达自由需要融入我们的代码中。隐私应该融入我们的代码中。普及所有知识。但是现在，这些值并没有嵌入到Web中。
>
> <cite>-Brewster Kahle，互联网档案馆的创始人（来自[锁定Web打开][brewster]）

## What are some use cases for WebTorrent?

WebTorrent最令人兴奋的用途之一是**同行辅助交付**. 非盈利项目[维基百科][wikipedia]以及[Internet存档][archive]可以通过让访问者插播来降低带宽和托管成本。流行的内容通过浏览器到浏览器提供服务，速度快且成本低。很少访问的内容通过HTTP从源服务器可靠地提供服务。

还有令人兴奋的**业务用例**从CDN到应用程序交付。

> WebTorrent具有巨大的业务潜力，可以通过内部基础设施和外部封闭用户通信的应用程序彻底改变客户机-服务器的传统概念。WebTorrent已经从一个“想法”变成了一个科学实验，现在正处于可行的边缘。这真的很酷。
>
> <cite>-Chris Kranky（来自[“WebTorrent:重新思考交付”][kranky-article]）</cite>

## How does WebTorrent work?

WebTorrent协议的工作原理与[BitTorrent协议][bittorrent-protocol]，除非它使用[韦伯特][webrtc]而不是[传输控制协议][tcp]/[uTP][utp]作为传输协议。

为了支持[WebRTC的连接模型][webrtc-signaling]我们对跟踪器协议做了一些修改。因此，基于浏览器的WebTorrent客户端或**“Web对等体”**只能连接到支持WebTorrent/WebRTC的其他客户端。

我们所做的协议更改将作为[BEP](http://www.bittorrent.org/beps/bep_0001.html). 在编写规范之前，可以查看[`bittorrent-tracker`][bittorrent-tracker]包裹。

一旦连接了对等端，用于通信的有线协议与正常BitTorrent中的协议完全相同。这应该使现有流行的Torrent客户机（如Transmission）和Autocurrent更容易添加对WebTorrent的支持。**武泽**
[已经得到支持][vuze-support]为了WebTorrent！

## How do I get started?

要开始使用WebTorrent，只需包括[`webtorrent.min.js`](https://cdn.jsdelivr.net/webtorrent/latest/webtorrent.min.js)在页面上编写脚本。如果你使用[browserify](http://browserify.org/)你可以`npm install webtorrent`和`require('webtorrent')`.

下载Torrent并将其添加到页面很容易。

```js
var client = new WebTorrent()

var torrentId = 'magnet:?xt=urn:btih:08ada5a7a6183aae1e09d831df6748d566095a10&dn=Sintel&tr=udp%3A%2F%2Fexplodie.org%3A6969&tr=udp%3A%2F%2Ftracker.coppersurfer.tk%3A6969&tr=udp%3A%2F%2Ftracker.empire-js.us%3A1337&tr=udp%3A%2F%2Ftracker.leechers-paradise.org%3A6969&tr=udp%3A%2F%2Ftracker.opentrackr.org%3A1337&tr=wss%3A%2F%2Ftracker.btorrent.xyz&tr=wss%3A%2F%2Ftracker.fastcast.nz&tr=wss%3A%2F%2Ftracker.openwebtorrent.com&ws=https%3A%2F%2Fwebtorrent.io%2Ftorrents%2F&xs=https%3A%2F%2Fwebtorrent.io%2Ftorrents%2Fsintel.torrent'

client.add(torrentId, function (torrent) {
  // Torrents can contain many files. Let's use the .mp4 file
  var file = torrent.files.find(function (file) {
    return file.name.endsWith('.mp4')
  })
  file.appendTo('body') // append the file to the DOM
})
```

它支持视频、音频、图像、PDF、降价，[更多][render-media]从盒子里拿出来。有其他直接访问文件内容的方法，包括作为节点样式的流、缓冲区或blob url。

可以传输视频和音频内容，即在下载完整文件之前开始播放。也在寻找工作——WebTorrent动态地从网络按需获取所需的Torrent片段。

## What is WebRTC?

WebRTC（Web实时通信）是由万维网联盟（W3C）定义的一个API，用于支持浏览器到浏览器的应用程序，如语音呼叫、视频聊天和P2P文件共享，而无需浏览器插件。

WebRTC`RTCDataChannel`API允许直接将数据从一个浏览器传输到另一个浏览器。这和`WebSocket`和`XMLHttpRequest`因为它们是为与服务器（即客户机-服务器模型）通信而设计的。数据通道允许**直接浏览器到浏览器的连接**.

这是革命性的。以前，网站从未能以超低延迟、加密的点对点连接直接将用户连接到彼此。这将在医疗、教育、科学等领域实现下一代应用。WebTorrent只是一个例子。

韦伯特[无处不在][webrtc-everywhere]和浏览器支持非常好。**铬**，**火狐**和**歌剧**对于桌面和Android，以及**微软边缘**和**游猎**有支持。

您可以在以下网址了解有关WebRTC数据通道的更多信息：[HTML5岩][datachannel-intro].

### A bit more about `webtorrent-hybrid`

在J.N.`webtorrent-hybrid`可以从WebRTC对等端或TCP对等端（即普通对等端）下载Torrent。您可以将WebTorrent用作命令行程序，或以编程方式用作node.js包。

安装`webtorrent-hybrid`在终端中运行以下命令（添加`-g`安装命令行程序的标志，省略该标志以在本地安装）：

```
npm install webtorrent-hybrid -g
```

注意：如果您只需要在浏览器中使用webtorrent（webrtc本机可用），则使用[`webtorrent`][webtorrent]相反，它的安装速度更快，因为它不需要安装WebRTC实现。

## Can WebTorrent clients on different websites connect to each other?

对！**WebTorrent在整个Web上工作。**在一个域上运行的WebTorrent客户端可以连接到任何其他域上的客户端。没有筒仓！

同一源策略不适用于WebRTC连接，因为它们不是客户端到服务器的连接。浏览器到浏览器的连接需要两个网站的合作（即，WebTorrent脚本必须同时出现在两个网站上）。

## How is WebTorrent different from PeerCDN?

[PeRCDN][peercdn]是由WebRTC支持的下一代cdn，用于高效地点对点交付网站内容。Peercdn由[阿巴克哈迪贾][feross]，[阿比拉贾][abi]和[约翰希西][jhiesey]2013年3月出售给[雅虎][yahoo]2013年12月。

WebTorrent是由[阿巴克哈迪贾][feross]2013年10月。不像PeerCDN，**WebTorrent是免费软件**，根据[MIT许可][license]. 你可以随心所欲地使用它！

> “免费软件”是自由的问题，而不是价格。要理解这个概念，你应该把“自由”看成是“言论自由”，而不是“啤酒自由”。
>
> <cite>-Richard Stallman，软件自由活动家</cite>

在技术层面上，peercdn和webtorrent的构建考虑了不同的目标。peercdn针对低延迟下载和快速对等发现进行了优化。这意味着客户端和站点所有者信任集中式服务器，可以将文件URL映射到内容哈希。

另一方面，WebTorrent不需要客户机信任集中式服务器。给定一个`.torrent`文件或磁铁链接，WebTorrent客户机下载文件时不信任任何服务器或对等机。

## WebTorrent supports sequential streaming. How does this affect the network?

BitTorrent客户机使用一种称为“最稀有优先”的算法来选择要下载的文件片段。系统中的每一个对等端都首先尝试下载最稀有的部分，平均来说，大多数部分在网络中的可用性将大致相同。

在实践中，最罕见的第一种算法对于种子不良的急流最为重要，或者在发布急流的前几个小时（当种子播种者与leechers的比率不好时）。

大多数Torrent客户机都支持一些特性，这些特性导致它偏离了一种最罕见的优先选择算法。例如，可以选择/取消选择或对Torrent中的某些文件进行优先级排序/降级。

WebTorrent支持按顺序传输Torrent文件，这对于播放媒体文件很有用。我们正致力于改进算法，在没有对特定部分的高优先级需求时，切换回最罕见的第一策略。换言之，当缓冲足够的介质时，我们可以使用普通的“最稀有的优先”工件选择算法。

但事实是，随着当今互联网连接的速度，用户将在使用所需时间的一小部分内完全下载完Torrent，因此他们仍将花费比下载更多的时间播种。

另请注意：BitTorrent公司的官方Torrent客户端utorrent提供顺序下载和选择性文件下载，BitTorrent网络仍然非常健康。

## Why wasn't WebTorrent designed as an entirely-new, modern protocol?

BitTorrent是目前最成功、部署最广泛的P2P协议。它真的很管用。我们使用WebTorrent的目标是以与现有Torrent网络互操作的方式将BitTorrent引入Web。

重新发明该协议将使WebTorrent从根本上与现有客户机不兼容，并阻止采用。我们这样做更好。有线协议是完全相同的，但是现在有了一种新的连接到对等端的方法：WebRTC，除了现有的TCP和UTP之外。

此外，重新发明协议是一个巨大的兔子洞。当我们开始这个项目的时候已经有了很大的风险——WebRTC会被所有的浏览器供应商所采用吗？数据通道实现是否稳定并得到执行？javascript是否足够快，可以动态重新打包MP4视频，以便使用MediaSource API进行流式播放？我们的想法是：为什么要在表格中添加一个新的有线协议和一些算法？

的确，BitTorrent协议在某些方面是过时的。例如，它使用自己的奇怪的数据编码，称为“bencoding”。如果它是今天发明的，它可能只使用JSON或messagepack。但是，这并不重要——BitTorrent工作得很好，我们更关心的是构建健壮有用的软件，而不是概念上的纯粹性或最新的软件时尚。

## Is it possible to do live streaming with WebTorrent?

WebTorrent不能进行开箱即用的实时流，但是您可以在WebTorrent之上构建实时流解决方案。

激流是不变的。这意味着一旦创建了一个Torrent文件，就不能在不更改信息哈希的情况下更改它。那么，怎样才能绕过这个限制呢？

一个幼稚的方法是：内容生产者可以每10秒获取一次实时内容并为其创建一个洪流。查看者将遵循这个“feed”的Torrent文件（或信息散列）并按顺序下载内容。拖缆将在实况流之后约10-20秒。

不过，这种方法肯定可以改进！为什么不自己试一试，分享代码呢？

## Does WebTorrent leak your IP address when using a VPN? I heard that WebRTC leaks your IP address.

不。

WebRTC数据通道不允许网站在使用VPN时发现您的公共IP地址。WebRTC发现过程只会找到您的VPN的IP地址和本地网络IP地址。

本地IP地址（例如10.x.x.x或192.168.x.x）可能用于“指纹”浏览器，并在您访问的不同站点之间进行识别，如第三方跟踪cookie。但是，这是一个与公开真实公共IP地址不同的问题，值得注意的是，浏览器已经为您提供了数百个指纹识别向量（例如，您安装的字体、屏幕分辨率、浏览器窗口大小、操作系统版本、语言等）。

如果您启用了VPN，那么WebRTC数据通道将不会使用您真正的公共IP地址连接到对等端，也不会显示在网页上运行的javascript中。

有一次，webrtc确实遇到了一个问题，它允许一个网站发现你真正的公共IP地址，但这个问题在很久以前就已经解决了。这种不幸的错误信息在互联网上不断反弹。

现在有了一个规范，它精确定义了WebRTC公开的IP地址。如果你对进一步阅读感兴趣，你可以阅读[IP handling spec](https://tools.ietf.org/html/draft-ietf-rtcweb-ip-handling-01)为你自己。

# Troubleshooting

## Why does browser downloading not work? I see no peers!

它确实有效！但你不能只使用任何随机磁铁uri或`.torrent`文件。Torrent必须由支持WebRTC的客户端（即[WebTorrent桌面][webtorrent-desktop]，[武泽][vuze-support]，[WebTorrent混合][webtorrent-hybrid]，[回放][playback]，[即时通讯][instant.io]或[β激流][btorrent].

在浏览器中，webtorrent只能通过支持webrtc的客户机下载明确播种到web对等机的torrent。桌面Torrent客户端需要支持WebRTC才能连接到Web浏览器。

## Why does video/audio streaming not work?

流支持取决于对`MediaSource`浏览器中的API。所有现代浏览器都有`MediaSource`支持。在firefox中，firefox 42增加了支持（即firefox nightly）。

[许多文件类型][render-media]支持（同样，取决于浏览器支持），但仅支持`.mp4`，`.m4v`和`.m4a`全力支持，包括寻求。

为了支持任意文件的视频/音频流，WebTorrent使用[`videostream`][videostream]包装，反过来使用[`mp4box.js`][mp4box.js]. 如果您认为其中一个包中可能存在错误，请在相应的存储库中提交一个问题。

[videostream]: https://npmjs.com/package/videostream

[mp4box.js]: https://github.com/gpac/mp4box.js
